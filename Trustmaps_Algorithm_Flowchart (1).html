Date Normalization Engine
Entry date ≠ display date. Always.
MVP
+
Rules
Trust

Flag

Reject

Burst Smoothing Engine

MVP
+
Rules
Detect
Burst trigger: Same contractor submits >3 jobs within 60 seconds → batch mode
Small
N ≤ 3 jobs: Release all immediately. No smoothing needed
Medium
N = 4–8 jobs: Release 3 now. Distribute rest over 72 hours, max 1 per 18–24h window

02 — Trust & Confirmation
💬
SMS Confirmation Engine
The trust backbone. Client-verified, not self-reported.
MVP
+
Rules
Timing
Send 24–48h after job date. Not immediately — homeowner calmer, payment settled
Format
One question only: "Did [Name] finish your [job type] this week? Reply YES or NO." No links, no login
Window
72-hour response window. One follow-up if no response. After that → no_response status
YES
YES path: Confirmed → optional photo prompt 10 min later → optional review 10 min after. Never forced
NO
NO path: Forwarded to Dispute Engine immediately. Pin hidden pending resolution
Limit
Same phone, same contractor: Max 1 confirmation per 60 days (90 days for seasonal services)
IP Check
Link opened from contractor IP: Confirmation voided silently. No warning to prevent workarounds
🔬
Information Analysis Engine
Validates photos, reviews, and data quality.
V2
+
Rules
Photo
Photos: Max 5/job. JPG/PNG/HEIC, 10MB, auto-compressed to 1200px. Blank/duplicate/low-res → rejected
Review
Review text: Max 280 chars. No URLs or phone numbers. Profanity filter flags, doesn't auto-reject
Fraud
Copy-paste review: Identical to prior submission → rejected. Same IP as contractor → voided
Attribution
All media labeled "by client" with job type and neighborhood. Client name never shown
Weight
Data quality bonus: Photo + confirmed review = +5pts · Photo only = +2pts · Review only = +2pts
Control
Contractor rights: Cannot delete client media. Can flag for review only
⚖️
Dispute Resolution Engine
Handles NO responses and job conflicts fairly.
MVP
+
Rules
First NO
Immediate action: Pin hidden from map. Contractor notified. 7-day resolution window opens
Resolve
Contractor resolves: Triggers new SMS to client. Confirmed → pin reappears with 'resolved' badge
Ignored
No response in 7 days: Pin stays hidden. Single isolated NO = zero score impact
Pattern
2+ NOs in 90 days: Account flagged. Visibility score capped at 60 until resolved manually
Suspend
3+ NOs in 90 days: Account suspended pending manual review
Protect
Competitor abuse: Denials only count if phone matches CRM record. Unmatched denials ignored
🛡️
Anti-Gaming Engine
Silent to honest contractors. Ruthless to bad actors.
V2
+
Rules
Phone
Uniqueness: Same phone confirms same contractor max 1× per 60 days
Match
Phone match required: Must match CRM record. No match = no badge, job still counted
Timestamp
Timestamp cluster: 5+ jobs with identical completion time (to the minute) → all held for review
Retroactive
Retroactive bomb: 10+ jobs in one session spread over 30 days → fraud flag raised
Farm
Confirmation farm: Same phone confirms 3+ different contractors in 24h → all flagged
VoIP
Temp/VoIP numbers: Carrier check on all phones. Flagged numbers need extra verification
New Acct
Account spike: Under 30 days old + 15+ jobs in week 1 → held for review
Score
Fraud score 0–100: 0–30 allow · 31–60 hold + notify · 61–80 suppress · 81+ suspend
03 — Scoring & Intelligence
⚡
Live Activity Score Engine
The master 0–100 score. Everything feeds this.
MVP
+
Rules
40%
Confirmed job volume: Last 14 days ×2 + days 15–30 ×1. Normalized to 40pts max
25%
Confirmation rate: Confirmed ÷ total submitted over 90 days. 80%+ = full 25pts. Linear below
20%
Recency: 0–3d=20pts · 4–7d=16 · 8–14d=11 · 15–21d=6 · 22–30d=2 · 30d+=0
10%
Hyperlocal signal: 3+ jobs within 1km in 30 days = 10pts · 2 jobs = 6pts · 1 = 2pts
5%
Client media: 2+ client photos = 5pts · 1 photo = 2pts
Freeze
Off-season override: Score frozen at last active value. No recalculation until season resumes
Floor
Score floor: Active subscribers never drop below 5 — always remain visible
⏳
Time Decay Engine
Past activity doesn't protect you forever.
MVP
+
Rules
Base
Base decay: −2.0 points per calendar day of zero activity during active season
Weekend
Weekend pause: Engine 07 shows contractor doesn't work weekends → decay pauses Sat + Sun
Batch
Batch-closer pause: After each batch submission, decay pauses 10 days to account for their cycle
Seasonal
Off-season freeze: Engine 08 returns off_season → decay = 0. Score completely frozen
Recovery
Recovery: Each confirmed job = +15pts + recency component fully reset
Dispute
Dispute acceleration: Active dispute → decay doubles to −4pts/day until resolved
🌿
Seasonal Intelligence Engine
Roofer in January ≠ plumber in January.
V2
+
Rules
PAI
Peer Activity Index: % of same-trade contractors in region with 1+ confirmed jobs in last 14 days
Active
AA ≥ EAI×0.5 + PAI>50%: Active season → normal scoring. No seasonal adjustment
Under
AA < EAI×0.5 + PAI>50%: Underperforming in active season → decay begins, pin greys out
Off-Season
AA=0 + PAI<30%: Most peers also quiet → seasonal mode after 14 days. Decay pauses
Resume
Reactivation: 2+ confirmed jobs → active mode resumes, score rebuilds from frozen value
Display
Badges: off_season = "❄️ Off-Season" · transition = "🌱 Season Starting Soon" · active = none
🧠
Contractor Behavior Engine
Learns each contractor's rhythm. Judges them on it.
V2
+
Rules
Pattern
Batch-closer detection: >60% of jobs entered on same weekday over 90 days → tag 'batch-closer:[day]'

Update
Recalculated: Every 7 days on rolling window. Profile always learning
